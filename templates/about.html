How It Works

  1. All incoming requests pass through the WAF middleware
  2. Requests are parsed and analyzed for malicious patterns
  3. IP address is checked against whitelist/blacklist
  4. Rate limiting is applied per IP address
  5. Pattern matching detects SQL injection, XSS, and path traversal
  6. Legitimate requests are allowed; malicious requests are blocked
  7. All events are logged for security monitoring

Configuration

The WAF can be configured through YAML and JSON files:

  • config/waf_config.yaml - Main configuration
  • config/waf_rules.json - Detection rules
  • config/whitelist.json - IP whitelist
